# Test Plan - SauceDemo E-Commerce Website

## Project Overview

This project focuses on testing the functionality of the SauceDemo e-commerce website. The purpose of testing is to verify that users can successfully log in, browse products, add items to the shopping cart, complete checkout, and log out.

## Application Under Test

Application:
SauceDemo

Website:
https://www.saucedemo.com

## Testing Objective

The objective of this testing project is to identify defects and verify that key user workflows function as expected.

## Scope of Testing

### Features Included:

- User Login
- Product Inventory
- Product Sorting
- Shopping Cart
- Checkout Process
- Logout

### Features Not Included:

- Backend testing
- Database testing
- Payment processing

## Testing Types

- Functional Testing
- Smoke Testing
- Regression Testing
- Exploratory Testing

## Test Environment

Browser:
Google Chrome

Operating System:
macOS

Testing Type:
Manual Testing

## Entry Criteria

Testing begins when:

- Application is available
- Test credentials are provided
- Testing environment is ready

## Exit Criteria

Testing is completed when:

- All planned test cases are executed
- Defects are documented
- Test summary is completed

## Defect Management

Defects will be documented with:

- Bug ID
- Description
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity
- Priority
